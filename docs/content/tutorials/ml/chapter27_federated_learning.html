Chapter 27. Federated Learning

Federated Learning is a distributed machine learning paradigm where multiple clients (e.g., mobile devices or edge servers) collaboratively train a model under the coordination of a central server, while keeping the training data decentralized. This approach enhances privacy and security by ensuring that raw data remains on local devices and only model updates are shared.

27.1. Federated Averaging (FedAvg) Algorithm

Federated Averaging (FedAvg) is a foundational algorithm in federated learning, designed to aggregate model updates from multiple clients to create a global model.

Key Concepts

  • Clients and Server: Federated learning involves multiple clients (e.g., devices with local data) and a central server that coordinates the training process.
    • Clients: Perform local training on their data and send model updates to the server.
    • Server: Aggregates the updates from all clients to form a global model.
  • Communication Rounds: The training process is divided into multiple rounds, where each round involves local training by clients and global aggregation by the server.

27.1.1. Federated Averaging Process

The FedAvg algorithm consists of the following steps:

  1. Initialization: The server initializes the global model with parameters \(\mathbf{w}_0\).

  2. Client Selection: In each round \(t\), a subset of clients is selected to participate in the training process.

  3. Local Training: Each selected client \(k\) performs local training on its dataset \(\mathcal{D}_k\) for multiple epochs, updating the local model parameters \(\mathbf{w}_k\).

  • Local Model Update: Clients update their model using standard optimization techniques (e.g., SGD). \[ \mathbf{w}_k^{t+1} = \mathbf{w}_k^t - \eta \nabla F_k(\mathbf{w}_k^t; \mathcal{D}_k) \] where \(\eta\) is the learning rate and \(\nabla F_k(\mathbf{w}_k^t; \mathcal{D}_k)\) is the gradient of the local objective function.

  1. Model Upload: After local training, clients send their updated model parameters \(\mathbf{w}_k^{t+1}\) to the server.

  2. Global Aggregation: The server aggregates the updates from all participating clients to form a new global model.

  • Federated Averaging: The server computes the average of the local model updates. \[ \mathbf{w}_{t+1} = \frac{1}{K} \sum_{k=1}^K \mathbf{w}_k^{t+1} \] where \(K\) is the number of participating clients.

  1. Broadcast Update: The server broadcasts the updated global model \(\mathbf{w}_{t+1}\) to all clients.

  2. Repeat: Steps 2-6 are repeated for multiple rounds until convergence.

Algorithm Pseudocode

Algorithm: Federated Averaging (FedAvg)

Input: Number of clients K, local epochs E, learning rate η, number of rounds T
Output: Global model parameters w

1. Initialize global model w0
2. For each round t = 0, 1, ..., T-1 do:
    a. Server selects a subset of clients S_t
    b. For each client k ∈ S_t in parallel do:
        i.    Client receives wt from server
        ii.   Client updates local model wk using local data D_k
              for epoch = 1, 2, ..., E do:
                  wk = wk - η ∇F_k(wk; D_k)
        iii.  Client sends updated model wk to server
    c. Server aggregates updates to form new global model:
        wt+1 = (1 / |S_t|) ∑ wk
    d. Server broadcasts wt+1 to all clients
3. Return final global model wT

Advantages of FedAvg

  • Privacy Preservation: By keeping raw data on local devices, FedAvg reduces the risk of data breaches and privacy violations.
  • Scalability: FedAvg can scale to a large number of clients, each with limited computational resources.
  • Communication Efficiency: By exchanging model updates rather than raw data, FedAvg reduces the communication overhead.

Challenges and Considerations

  • Non-IID Data: Client data may be non-IID (non-independent and identically distributed), which can affect the convergence and performance of the global model.
  • Client Availability: Clients may have intermittent availability, impacting the consistency and reliability of the training process.
  • Resource Constraints: Clients may have varying computational and storage capabilities, requiring adaptive strategies to handle heterogeneous environments.

By leveraging the Federated Averaging algorithm, federated learning enables collaborative model training across decentralized data sources, enhancing privacy and enabling large-scale learning applications in diverse and distributed settings.

27.2. Privacy-preserving Machine Learning

Privacy-preserving machine learning aims to protect the confidentiality and integrity of data during the training and inference processes. This is particularly important in federated learning, where sensitive data remains distributed across multiple clients.

27.2.1. Differential Privacy in Federated Learning

Differential privacy provides a formal framework to ensure that the inclusion or exclusion of a single data point does not significantly affect the output of an analysis, thus protecting individual privacy.

Key Concepts:

  • Differential Privacy Definition: A mechanism \(\mathcal{M}\) is \(\epsilon\)-differentially private if for any two neighboring datasets \(D\) and \(D'\) differing by at most one element, and for any possible output \(S\) of \(\mathcal{M}\): \[ \Pr[\mathcal{M}(D) = S] \leq e^\epsilon \Pr[\mathcal{M}(D') = S] \]

  • Mechanisms for Differential Privacy:

    • Noise Addition: Adding noise to the model updates to mask the contribution of individual data points. \[ \tilde{\mathbf{w}}_k^{t+1} = \mathbf{w}_k^{t+1} + \mathcal{N}(0, \sigma^2) \] where \(\mathcal{N}(0, \sigma^2)\) is Gaussian noise with variance \(\sigma^2\).

  • Implementation in Federated Learning:

    • Local Differential Privacy: Clients add noise to their updates before sending them to the server.
    • Global Differential Privacy: The server adds noise during the aggregation of updates.

Advantages:

  • Protects individual data points from being inferred from the model updates.
  • Provides a quantifiable privacy guarantee.

Disadvantages:

  • Can reduce the accuracy of the model due to the added noise.
  • Requires careful tuning of the noise parameter \(\epsilon\).

27.2.2. Secure Aggregation Protocols

Secure aggregation ensures that the server can aggregate model updates from clients without learning the individual updates, thereby enhancing privacy.

Key Concepts:

  • Encryption: Each client encrypts its model update before sending it to the server. \[ \mathbf{c}_k = \text{Enc}(\mathbf{w}_k^{t+1}) \]

  • Aggregation: The server aggregates the encrypted updates. \[ \mathbf{c} = \sum_{k=1}^K \mathbf{c}_k \]

  • Decryption: The aggregated update is decrypted to obtain the global model update. \[ \mathbf{w}_{t+1} = \text{Dec}(\mathbf{c}) \]

  • Protocols:

    • Secure Multiparty Computation (SMC): Clients collaboratively compute the aggregation function without revealing their individual updates.
    • Homomorphic Encryption: Allows the server to perform computations on encrypted data without needing to decrypt it.

Advantages:

  • Enhances privacy by ensuring the server cannot access individual updates.
  • Maintains the confidentiality of client data during the aggregation process.

Disadvantages:

  • Can introduce significant computational overhead.
  • Requires complex cryptographic techniques and careful implementation.

27.2.3. Homomorphic Encryption in Federated Learning

Homomorphic encryption allows computations to be performed on encrypted data, enabling secure federated learning without revealing individual model updates.

Key Concepts:

  • Homomorphic Encryption: A form of encryption that allows specific types of computations to be carried out on ciphertext, generating an encrypted result that, when decrypted, matches the result of operations performed on the plaintext.

Process:

  • Encryption: Clients encrypt their model updates before sending them to the server. \[ \mathbf{c}_k = \text{HE.Enc}(\mathbf{w}_k^{t+1}) \]

  • Aggregation on Encrypted Data: The server aggregates the encrypted updates. \[ \mathbf{c}_{\text{agg}} = \sum_{k=1}^K \mathbf{c}_k \]

  • Decryption: The aggregated encrypted update is decrypted by an authorized entity to obtain the global model update. \[ \mathbf{w}_{t+1} = \text{HE.Dec}(\mathbf{c}_{\text{agg}}) \]

Advantages:

  • Provides strong security guarantees by keeping data encrypted during computation.
  • Protects the privacy of individual model updates.

Disadvantages:

  • Computationally expensive and resource-intensive.
  • Complex implementation and potential scalability issues.

Algorithm: Differential Privacy in Federated Learning

Input: Number of clients K, local epochs E, learning rate η, noise scale σ, number of rounds T Output: Global model parameters w

  1. Initialize global model w0
  2. For each round t = 0, 1, …, T-1 do:
    1. Server selects a subset of clients S_t
    2. For each client k ∈ S_t in parallel do:
      1. Client receives wt from server
      2. Client updates local model wk using local data D_k for epoch = 1, 2, …, E do: wk = wk - η ∇F_k(wk; D_k)
      3. Client adds noise: wk = wk + N(0, σ^2)
      4. Client sends noisy model wk to server
    3. Server aggregates updates to form new global model: wt+1 = (1 / |S_t|) ∑ wk
    4. Server broadcasts wt+1 to all clients
  3. Return final global model wT

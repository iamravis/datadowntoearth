Generative Modeling for Text

Language Models and Next-Word Prediction

Language models are fundamental to generative text modeling. They learn to predict the probability distribution of the next word given the previous words in a sequence. This ability to predict the next word in a sequence is crucial for tasks such as text generation, machine translation, and speech recognition.

Probabilistic Formulation

Given a sequence of words \(w_1, w_2, \ldots, w_n\), a language model computes the joint probability of the entire sequence as the product of conditional probabilities of each word given the preceding words:

\[ P(w_1, w_2, \ldots, w_n) = \prod_{i=1}^n P(w_i | w_1, \ldots, w_{i-1}) \]

This decomposition is based on the chain rule of probability, which expresses the joint probability of a sequence in terms of conditional probabilities.

N-gram Models

N-gram models approximate the probability of a word given its preceding words by considering only the previous \(n-1\) words:

\[ P(w_i | w_1, \ldots, w_{i-1}) \approx P(w_i | w_{i-n+1}, \ldots, w_{i-1}) \]

Characteristics of N-gram Models

  • Simplification: By limiting the context to the last \(n-1\) words, N-gram models simplify the prediction problem.
  • Data Sparsity: Higher-order N-grams (larger \(n\)) suffer from data sparsity, as the probability estimates become unreliable due to insufficient data.
  • Smoothing Techniques: To address data sparsity, techniques like Laplace smoothing, Good-Turing smoothing, and Kneser-Ney smoothing are employed to adjust the probability estimates.

Neural Language Models

Neural language models use neural networks to learn more complex representations of context, allowing them to capture long-range dependencies and nuanced patterns in the data.

Common Architecture

  1. Embedding Layer: Maps words to dense vector representations, capturing syntactic and semantic information.
  2. Hidden Layers: Process the context using recurrent layers (e.g., LSTM, GRU) or attention mechanisms (e.g., Transformers).
  3. Output Layer: Produces a probability distribution over the vocabulary, often implemented as a softmax layer.

The model is typically trained to minimize the negative log-likelihood of the predicted words given the context:

\[ \mathcal{L} = -\sum_{i=1}^n \log P(w_i | w_1, \ldots, w_{i-1}) \]

Examples of Neural Language Models

  1. Recurrent Neural Networks (RNNs): Process sequences by maintaining a hidden state that captures information about the previous words.
  2. Long Short-Term Memory (LSTM) Networks: Address the vanishing gradient problem in RNNs by introducing gating mechanisms.
  3. Transformer Models: Use self-attention mechanisms to process sequences in parallel, improving efficiency and capturing long-range dependencies.

Sequence-to-Sequence Models

Sequence-to-sequence (seq2seq) models are designed to transform an input sequence into an output sequence. They are widely used in tasks like machine translation, summarization, and dialogue generation.

General Architecture

A seq2seq model consists of two main components:

  1. Encoder: Processes the input sequence and produces a context representation, often a fixed-length vector.
  2. Decoder: Generates the output sequence based on the context from the encoder.

Encoder-Decoder Interaction

  • Attention Mechanism: Enhances the basic seq2seq architecture by allowing the decoder to focus on different parts of the input sequence at each decoding step.
  • Bidirectional RNNs: Improve the encoder by processing the input sequence in both forward and backward directions.

RNNs (Recurrent Neural Networks)

RNNs process sequences by maintaining a hidden state that is updated at each time step:

\[ h_t = f(h_{t-1}, x_t) \]

Where \(h_t\) is the hidden state at time \(t\), \(x_t\) is the input at time \(t\), and \(f\) is a non-linear function (e.g., tanh or ReLU).

The output at each time step is typically computed as:

\[ y_t = g(h_t) \]

Where \(g\) is another non-linear function.

LSTMs (Long Short-Term Memory)

LSTMs address the vanishing gradient problem in RNNs by introducing gating mechanisms to control the flow of information:

  1. Forget Gate: Decides what information to discard from the cell state:

\[ f_t = \sigma(W_f \cdot [h_{t-1}, x_t] + b_f) \]

  1. Input Gate: Decides what new information to store in the cell state:

\[ i_t = \sigma(W_i \cdot [h_{t-1}, x_t] + b_i) \]

  1. Output Gate: Decides what information to output from the cell state:

\[ o_t = \sigma(W_o \cdot [h_{t-1}, x_t] + b_o) \]

  1. Cell State Update: Combines the old cell state with new candidate values:

\[ C_t = f_t * C_{t-1} + i_t * \tanh(W_C \cdot [h_{t-1}, x_t] + b_C) \]

  1. Hidden State Update: Produces the final hidden state for the current time step:

\[ h_t = o_t * \tanh(C_t) \]

Where \(\sigma\) is the sigmoid function and \(*\) denotes element-wise multiplication.

Transformers

Transformers use self-attention mechanisms to process sequences in parallel, allowing for more efficient training on large datasets.

Key Components

  1. Multi-Head Attention: Allows the model to attend to different parts of the input simultaneously, enhancing its ability to capture various dependencies:

\[ \text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V \]

Where \(Q\), \(K\), and \(V\) are query, key, and value matrices derived from the input, and \(d_k\) is the dimension of the key vectors.

  1. Positional Encoding: Injects information about the position of tokens in the sequence, enabling the model to capture order information despite processing tokens in parallel.

  2. Feed-Forward Networks: Process the outputs of attention layers, typically implemented as two linear transformations with a ReLU activation in between.

Transformer Architecture

  • Encoder: Consists of multiple layers, each containing a multi-head attention mechanism and a feed-forward network.
  • Decoder: Similar to the encoder but includes an additional attention layer to attend to the encoder’s output.

Beam Search and Sampling Techniques

When generating text, selecting the most appropriate words at each step is crucial. Common techniques include:

Greedy Search

At each step, choose the word with the highest probability:

\[ w_t = \arg\max_w P(w | w_1, \ldots, w_{t-1}) \]

Advantages and Disadvantages

  • Advantages: Simple and fast.
  • Disadvantages: Often leads to suboptimal solutions, as it may get stuck in local maxima.

Beam Search

Maintains \(k\) best partial hypotheses at each step. The beam search score for a sequence is typically:

\[ \text{score}(w_1, \ldots, w_t) = \sum_{i=1}^t \log P(w_i | w_1, \ldots, w_{i-1}) \]

Steps in Beam Search

  1. Initialization: Start with a set of initial hypotheses, typically the start token.
  2. Expansion: At each time step, expand each hypothesis by generating all possible next words.
  3. Scoring: Calculate the score for each expanded hypothesis.
  4. Pruning: Keep the top \(k\) hypotheses based on their scores.
  5. Termination: Continue until all hypotheses reach the end token or a predefined length.

Top-k Sampling

Sample from the top \(k\) most probable words at each step:

  1. Sort the vocabulary by probability.
  2. Keep only the top \(k\) words.
  3. Renormalize the probabilities.
  4. Sample from this truncated distribution.

Nucleus (Top-p) Sampling

Sample from the smallest set of words whose cumulative probability exceeds a threshold \(p\):

  1. Sort the vocabulary by probability.
  2. Find the smallest set of words whose cumulative probability exceeds \(p\).
  3. Renormalize the probabilities of this set.
  4. Sample from this dynamic distribution.

Generating Coherent and Diverse Text

Generating high-quality text involves balancing coherence and diversity. Techniques to achieve this include:

  1. Temperature Scaling: Adjust the “sharpness” of the output distribution:

\[ P_{\text{temp}}(w_i) = \frac{\exp(z_i / T)}{\sum_j \exp(z_j / T)} \]

Where \(z_i\) are the logits and \(T\) is the temperature parameter. Lower \(T\) makes the distribution peakier, higher \(T\) makes it more uniform.

  1. Repetition Penalties: Reduce the probability of repeating words or phrases by adjusting the logits before sampling.

  2. Length Normalization: Adjust scores based on sequence length to avoid bias towards shorter sequences:

\[ \text{score}_{\text{norm}}(y_1, \ldots, y_t) = \frac{\text{score}(y_1, \ldots, y_t)}{t^\alpha} \]

Where \(\alpha\) is a hyperparameter (typically between 0 and 1).

  1. Diversity-Promoting Objectives: Encourage the model to generate diverse outputs by incorporating additional terms in the loss function that penalize similarity between generated sequences.

Techniques for Improving Text Generation

Attention Mechanisms

Attention allows models to focus on relevant parts of the input when generating each output token. The attention weight \(\alpha_{ij}\) between the \(i\)-th output and \(j\)-th input is typically computed as:

\[ \alpha_{ij} = \frac{\exp(e_{ij})}{\sum_k \exp(e_{ik})} \]

Where \(e_{ij}\) is a score measuring the compatibility of the \(i\)-th output with the \(j\)-th input.

Types of Attention Mechanisms

  • Scaled Dot-Product Attention: Computes attention using dot products of queries and keys, scaled by the square root of the dimension of the keys.
  • Multi-Head Attention: Uses multiple attention heads to capture different aspects of the input.

Pointer-Generator Networks

Pointer-generator networks combine the ability to generate words from a fixed vocabulary with the ability to copy words directly from the input. The probability of generating a word \(w\) is:

\[ P(w) = p_{gen} P_{vocab}(w) + (1 - p_{gen}) \sum_{i:w_i=w} \alpha_i \]

Where \(p_{gen}\) is the probability of generating from the vocabulary, \(P_{vocab}(w)\) is the probability of generating \(w\) from the vocabulary, and \(\alpha_i\) is the attention weight on the \(i\)-th input word.

Applications

  • Text Summarization: Generates summaries by combining generated and copied content.
  • Question Answering: Produces answers by copying relevant information from the input context.

Reinforcement Learning

Reinforcement learning can be used to directly optimize for task-specific metrics that are non-differentiable. The REINFORCE algorithm updates the model parameters \(\theta\) as:

\[ \nabla_\theta J(\theta) \approx (R(y) - b) \nabla_\theta \log P(y|\theta) \]

Where \(R(y)\) is the reward for the generated sequence \(y\), and \(b\) is a baseline to reduce variance.

Policy Gradient Methods

  • REINFORCE: A basic policy gradient method for optimizing the expected reward.
  • Actor-Critic: Combines policy gradient (actor) with a value function estimator (critic) to reduce variance and improve stability.

Other Advanced Techniques

  1. Curriculum Learning: Start with easier tasks and gradually increase difficulty during training. This helps the model learn more effectively by building on simpler tasks.
  2. Transfer Learning: Pre-train on large corpora and fine-tune on specific tasks. This leverages the knowledge gained from pre-training to improve performance on specialized tasks.
  3. Adversarial Training: Use discriminator networks to provide feedback on generated text. The generator and discriminator compete, improving the quality of generated text.
  4. Controllable Text Generation: Condition the generation on specific attributes or styles. This allows for generating text with desired characteristics, such as sentiment or formality.
  5. Few-shot Learning: Adapt to new tasks with minimal examples. This involves training models to generalize from a few examples, making them more flexible and adaptable.
  6. Prompt Engineering: Design effective prompts to guide the generation process. Carefully crafted prompts can significantly influence the quality and relevance of generated text.

Beam Search

Beam search is a heuristic search algorithm that explores a graph by expanding the most promising node in a limited set. In the context of text generation, it’s used to find the most likely sequence of words.

Basic Beam Search Algorithm

  1. Initialization: Start with an initial set of partial sequences (usually just the start token).
  2. Expansion: For each time step:
    • For each partial sequence in the beam, generate the next possible tokens and their probabilities.
    • Calculate the score for each new possible sequence.
    • Select the top-\(k\) sequences based on their scores, where \(k\) is the beam width.
  3. Termination: Continue until a stopping criterion is met (e.g., max length or generation of the end token).

The score for a sequence is typically the log-probability of the sequence:

\[ \text{score}(y_1, \ldots, y_t) = \sum_{i=1}^t \log P(y_i | y_1, \ldots, y_{i-1}, x) \]

Where \(x\) is the input (e.g., for translation tasks) and \(y_i\) are the generated tokens.

Length Normalization

To avoid bias towards shorter sequences, length normalization can be applied:

\[ \text{score}_{\text{norm}}(y_1, \ldots, y_t) = \frac{\text{score}(y_1, \ldots, y_t)}{t^\alpha} \]

Where \(\alpha\) is a hyperparameter (typically between 0 and 1).

Diverse Beam Search

To generate diverse outputs, diverse beam search introduces a diversity penalty:

\[ \text{score}_{\text{diverse}}(y_1, \ldots, y_t) = \text{score}(y_1, \ldots, y_t) - \lambda \cdot \text{diversity}(y_1, \ldots, y_t) \]

Where \(\lambda\) is a hyperparameter and diversity() measures the similarity to previously generated sequences.

Sampling Techniques

Sampling introduces randomness into the generation process, potentially leading to more diverse and creative outputs.

Top-k Sampling

  1. Sort the vocabulary by probability.
  2. Keep only the top \(k\) words.
  3. Renormalize the probabilities.
  4. Sample from this truncated distribution.

Nucleus (Top-p) Sampling

  1. Sort tokens by probability.
  2. Select the smallest set of tokens whose cumulative probability exceeds \(p\).
  3. Renormalize probabilities within this set.
  4. Sample from this dynamic distribution.

The probability mass is calculated as:

\[ P_{\text{nucleus}}(w) = \begin{cases} \frac{P(w)}{\sum_{w' \in V_p} P(w')} & \text{if } w \in V_p \\ 0 & \text{otherwise} \end{cases} \]

Where \(V_p\) is the set of top tokens whose cumulative probability exceeds \(p\).

Temperature Sampling

Adjust the “sharpness” of the probability distribution:

\[ P_{\text{temp}}(w_i) = \frac{\exp(z_i / T)}{\sum_j \exp(z_j / T)} \]

Where \(z_i\) are the logits and \(T\) is the temperature parameter. Lower \(T\) makes the distribution peakier, higher \(T\) makes it more uniform.

Advanced Techniques

Stochastic Beam Search

Stochastic Beam Search combines the benefits of beam search and sampling. It uses the Gumbel-Max trick to sample without replacement:

  1. Add Gumbel Noise to the log-probabilities: \(\tilde{s}_i = \log P(w_i) + g_i\), where \(g_i \sim \text{Gumbel}(0, 1)\).
  2. Select the top-\(k\) sequences based on these perturbed scores.

This allows for exploration of lower-probability sequences while maintaining the efficiency of beam search.

Constrained Beam Search

Constrained Beam Search allows for the incorporation of hard constraints on the output:

  1. Define a set of constraints (e.g., inclusion of specific words, adherence to a grammar).
  2. Expand sequences that satisfy or can potentially satisfy the constraints.
  3. Use a constraint satisfaction algorithm to guide the search.

This is particularly useful for tasks like constrained text generation or grammar-guided parsing.

Adaptive Beam Search

Adaptive Beam Search dynamically adjusts the beam size based on the current state of the search:

  1. Start with an initial beam size.
  2. Evaluate the quality and diversity of the current beam.
  3. Increase the beam size if the quality is low or diversity is high.
  4. Decrease the beam size if the quality is high and diversity is low.

This can lead to more efficient search, especially for variable-length outputs.

Iterative Refinement

Iterative Refinement techniques generate an initial sequence and then iteratively improve it:

  1. Generate an initial sequence using any method (e.g., greedy search).
  2. Identify parts of the sequence that need improvement.
  3. Regenerate those parts while keeping the rest fixed.
  4. Repeat steps 2-3 for a fixed number of iterations or until convergence.

This can be particularly effective for tasks like machine translation or text summarization.

Reinforcement Learning for Sequence Generation

Reinforcement Learning (RL) can be used to directly optimize for task-specific metrics:

  1. Markov Decision Process: Treat the generation process as a sequence of decisions.
  2. Reward Function: Define a reward function based on the desired metric (e.g., BLEU score for translation).
  3. RL Algorithms: Use RL algorithms (e.g., REINFORCE, Actor-Critic) to optimize the model.

The REINFORCE algorithm updates the model parameters \(\theta\) as:

\[ \nabla_\theta J(\theta) \approx (R(y) - b) \nabla_\theta \log P(y|\theta) \]

Where \(R(y)\) is the reward for the generated sequence \(y\), and \(b\) is a baseline to reduce variance.

These advanced techniques allow for more controlled, diverse, and task-specific text generation, addressing many of the limitations of basic beam search and sampling methods.

Chapter 8: Generative Modeling For Tabular Data

Generative modeling for tabular data involves creating synthetic datasets that maintain the statistical properties and relationships of the original data. This is particularly challenging due to the structured nature of tabular data and the complex dependencies between features.

Modeling Structured Data with Dependencies

Tabular data often contains a mix of continuous, categorical, and ordinal variables with complex relationships. Modeling these dependencies is crucial for generating realistic synthetic data.

Copulas

Copulas are a powerful tool for modeling multivariate distributions and capturing dependencies between variables:

\[ C(u_1, ..., u_d) = P(U_1 \leq u_1, ..., U_d \leq u_d) \]

Where \(U_i\) are uniform random variables. The Sklar’s theorem states that any multivariate distribution can be expressed in terms of a copula and marginal distributions:

\[ F(x_1, ..., x_d) = C(F_1(x_1), ..., F_d(x_d)) \]

Types of Copulas

  1. Gaussian Copula: Based on the multivariate normal distribution.

    \[ C_{\Sigma}(u_1, ..., u_d) = \Phi_{\Sigma}(\Phi^{-1}(u_1), ..., \Phi^{-1}(u_d)) \]

    Where \(\Phi_{\Sigma}\) is the multivariate normal CDF with correlation matrix \(\Sigma\), and \(\Phi^{-1}\) is the inverse of the standard normal CDF.

  2. Archimedean Copulas: A family of copulas defined by a generator function \(\psi\).

    \[ C(u_1, ..., u_d) = \psi^{-1}(\psi(u_1) + ... + \psi(u_d)) \]

  3. Vine Copulas: A flexible way to construct high-dimensional copulas using a cascade of bivariate copulas.

Bayesian Networks

Bayesian networks represent conditional dependencies between variables using a directed acyclic graph (DAG). The joint distribution is factorized as:

\[ P(X_1, ..., X_n) = \prod_{i=1}^n P(X_i | \text{Parents}(X_i)) \]

Learning Bayesian Networks

  1. Structure Learning: Determining the DAG structure.
    • Score-based methods: Optimize a scoring function (e.g., BIC, AIC).
    • Constraint-based methods: Use conditional independence tests.
  2. Parameter Learning: Estimating the conditional probability distributions.
    • Maximum Likelihood Estimation (MLE)
    • Bayesian estimation with prior distributions

Inference in Bayesian Networks

  1. Exact Inference: Methods like Variable Elimination and Junction Tree algorithm.
  2. Approximate Inference: Methods like Markov Chain Monte Carlo (MCMC) and Variational Inference.

Generative Adversarial Networks for Tabular Data

GANs have been adapted for tabular data generation, addressing challenges such as mixed data types and mode collapse.

TableGAN

TableGAN introduces several modifications to the standard GAN architecture for tabular data:

  1. Conditional Generation: The generator is conditioned on class labels to improve diversity.

    \[ G(z, c) = g_{\theta}(z, c) \]

    Where \(z\) is the noise input and \(c\) is the class label.

  2. Mode-Specific Normalization: Applies different normalization for continuous and categorical variables.

    For continuous variables: \[ \hat{x} = \frac{x - \mu}{\sigma} \]

    For categorical variables: \[ \hat{x} = \text{softmax}(x) \]

  3. Information Loss: Introduces an additional loss term to preserve feature-wise correlations.

    \[ \mathcal{L}_{info} = ||Corr(X_{real}) - Corr(X_{gen})||_F \]

    Where \(Corr(\cdot)\) is the correlation matrix and \(||\cdot||_F\) is the Frobenius norm.

The objective function includes the standard GAN loss plus an information loss:

\[ \mathcal{L} = \mathcal{L}_{GAN} + \lambda \mathcal{L}_{info} \]

CTGAN (Conditional Tabular GAN)

CTGAN addresses the challenges of imbalanced categorical columns and mixed data types:

  1. Conditional Vector: Incorporates a conditional vector to handle categorical columns.

    \[ c = [c_1, ..., c_m], \quad c_i \in \{0, 1\}^{d_i} \]

    Where \(m\) is the number of categorical columns and \(d_i\) is the number of categories in the \(i\)-th column.

  2. Mode-Specific Normalization: Similar to TableGAN, applies different normalization for different data types.

  3. Training-by-Sampling: Balances the training of rare categories by using a sampling technique.

    \[ P(c_i = k) = \frac{\log(f_{i,k} + 1)}{\sum_{j=1}^{d_i} \log(f_{i,j} + 1)} \]

    Where \(f_{i,k}\) is the frequency of the \(k\)-th category in the \(i\)-th column.

The generator’s input includes noise \(z\), conditional vector \(c\), and mask vector \(m\):

\[ G(z, c, m) = g_{\theta}(z, c) \odot (1-m) + c \odot m \]

Where \(g_{\theta}\) is the generator network and \(\odot\) is element-wise multiplication.

Variational Autoencoders for Tabular Data

VAEs have been adapted for tabular data generation, offering a different approach to capturing data distributions.

TVAE (Tabular Variational Autoencoder)

TVAE extends the VAE framework to handle tabular data:

  1. Mixed Data Types: Uses different output layers for continuous and categorical variables.

    For continuous variables: \[ p(x_i|z) = \mathcal{N}(\mu_i(z), \sigma_i^2(z)) \]

    For categorical variables: \[ p(x_i|z) = \text{Cat}(\pi_i(z)) \]

  2. Conditional Generation: Incorporates conditional information to control generated samples.

    \[ q_{\phi}(z|x, c) = \mathcal{N}(\mu_{\phi}(x, c), \sigma_{\phi}^2(x, c)) \]

  3. Disentangled Representation: Encourages learning of independent latent factors.

    \[ \mathcal{L}_{disentangle} = \sum_{i \neq j} |\text{Corr}(z_i, z_j)| \]

The TVAE objective function includes the standard VAE loss plus a disentanglement term:

\[ \mathcal{L} = \mathcal{L}_{VAE} + \beta \mathcal{L}_{disentangle} \]

Where \(\beta\) controls the strength of the disentanglement regularization.

Synthetic Data Generation and Augmentation

Synthetic data generation for tabular data serves several purposes:

  1. Data Augmentation: Increasing dataset size to improve model performance.
  2. Privacy Preservation: Generating synthetic data that maintains statistical properties without exposing individual records.
  3. Imbalanced Data Handling: Generating synthetic samples for underrepresented classes.

Techniques for synthetic data generation include:

SMOTE (Synthetic Minority Over-sampling Technique)

SMOTE generates synthetic samples by interpolating between existing minority class samples:

  1. For each minority class sample \(x_i\), find its \(k\) nearest neighbors.

  2. Choose one of these neighbors randomly, say \(x_j\).

  3. Create a synthetic sample:

    \[ x_{new} = x_i + \lambda(x_j - x_i) \]

    Where \(\lambda \in [0, 1]\) is a random number.

DataSynthesizer

DataSynthesizer uses Bayesian networks to capture variable dependencies and generate synthetic data:

  1. Learn the Bayesian network structure from the data.
  2. Estimate the conditional probability distributions for each node.
  3. Generate synthetic data by sampling from the learned Bayesian network.

SDV (Synthetic Data Vault)

SDV is a framework that includes multiple techniques for synthetic data generation, including GAN and VAE-based methods. It provides:

  1. Data modeling: Capture the statistical properties of the original data.
  2. Synthetic data generation: Create new, realistic synthetic samples.
  3. Quality evaluation: Assess the similarity between real and synthetic data.

Evaluation Metrics for Tabular Data Generation

Evaluating the quality of generated tabular data is crucial. Common metrics include:

Statistical Similarity

  1. Kolmogorov-Smirnov Test: Measures the similarity of continuous variable distributions:

    \[ D_{KS} = \sup_x |F_1(x) - F_2(x)| \]

    Where \(F_1\) and \(F_2\) are the cumulative distribution functions of real and synthetic data.

  2. Chi-Square Test: Assesses the similarity of categorical variable distributions.

    \[ \chi^2 = \sum_{i=1}^k \frac{(O_i - E_i)^2}{E_i} \]

    Where \(O_i\) are observed frequencies and \(E_i\) are expected frequencies.

Machine Learning Efficacy

  1. Classification Performance Delta: Difference in performance of a classifier trained on real vs. synthetic data:

    \[ \Delta_{perf} = |Perf_{real} - Perf_{synthetic}| \]

  2. Cross-Fitting: Train on synthetic, test on real (and vice versa), compare performance.

    \[ \text{Cross-Fit Score} = \frac{1}{2}(Perf_{train\_syn\_test\_real} + Perf_{train\_real\_test\_syn}) \]

Privacy Metrics

  1. Nearest Neighbor Distance Ratio: Ratio of distances to nearest neighbors in real and synthetic data:

    \[ NNDR = \frac{\min_{y \in \text{Synthetic}} d(x, y)}{\min_{x' \in \text{Real}, x' \neq x} d(x, x')} \]

  2. Membership Inference Attack: Measure the ability to determine if a record was in the training data.

    \[ \text{Privacy Risk} = \frac{\text{True Positives} + \text{True Negatives}}{\text{Total Samples}} \]

Correlation Preservation

  1. Pairwise Correlation Difference: Measures how well the synthetic data preserves feature correlations:

    \[ \Delta_{corr} = \sum_{i,j} |Corr_{real}(X_i, X_j) - Corr_{synthetic}(X_i, X_j)| \]

  2. Mutual Information Preservation: Compares mutual information between features in real and synthetic data:

    \[ \Delta_{MI} = \sum_{i,j} |MI_{real}(X_i, X_j) - MI_{synthetic}(X_i, X_j)| \]

These evaluation metrics help ensure that the generated tabular data maintains the statistical properties, relationships, and utility of the original data while preserving privacy and addressing specific use-case requirements.

The field of generative modeling for tabular data is rapidly evolving, with new techniques and improvements being developed to address the unique challenges posed by structured data. As these methods continue to advance, they promise to provide valuable tools for data augmentation, privacy preservation, and synthetic data generation across a wide range of applications.
